Status: Done

## Acceptance Criteria

- [x] UI shows generated cabinets in a scannable table/list.
- [x] Designer can edit width, height, depth, type/name, and notes.
- [x] Invalid edits show validation errors before saving.
- [x] Cabinet count and total occupied length update after edits.
- [x] Edited cabinet list persists on the design session.
- [x] Changes are not pushed to BOM/Fusion automatically.

## Implementation Notes

- The measured layout page now renders a cabinet table after generation.
- Editable fields are:
  - label/name;
  - category/type;
  - width, height, and depth;
  - notes.
- Saving calls `PUT /api/design-sessions/{sessionId}/cabinet-plan`.
- The backend validates the submitted list with the deterministic rules from #43.
- If validation returns errors, the response is returned with `valid=false` and the plan is not persisted.
- If validation passes, the backend updates the `Cabinet` records and refreshes the inline `cabinet-plan.json` design artifact.
- No BOM, Fusion, quote, or manufacturing artifact is generated by this step.

## Test Notes

- Integration tests cover valid manual edits persisting to `Cabinet`.
- Integration tests cover invalid manual edits returning structured validation messages without replacing the persisted plan.
